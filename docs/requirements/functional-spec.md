# 功能规格说明书 (Functional Specification)

## 文档信息

- **项目**：zen-gomoku（禅五子棋）
- **版本**：v0.1（核心对局）
- **最后更新**：2026-03-02

---

## 1. 引言

### 1.1 目的

本文档描述 zen-gomoku 项目的功能需求，作为开发与验收的依据。

### 1.2 范围

- **产品**：15×15 标准五子棋，支持人人对战。
- **平台**：PC Web、Mobile H5、微信内置浏览器；交互需统一处理 Mouse 与 Touch。
- **AI**：Phase 1 随机/规则；Phase 2 Minimax+AlphaBeta；Phase 3 预留 AlphaZero 接口。
- **棋谱**：支持保存/加载（SGF 或自定义 JSON），支持复盘。
- **小程序**：架构预留微信小程序/小游戏适配（逻辑与视图分离）。

### 1.3 定义与缩写

| 术语     | 说明                                                     |
| -------- | -------------------------------------------------------- |
| 五子棋   | 连五子（横竖斜）即胜                                     |
| 人人对战 | 两名人类玩家轮流落子                                     |
| 复盘     | 按步数回放对局                                           |
| 规则     | v0.1 采用**无禁手**（无三三、四四、长连禁手）            |
| 和棋     | 棋盘满且无五连判和棋；若 v0.1 不实现和棋则仅一方胜即结束 |

## 2. 功能需求

### 2.1 核心玩法

| 编号      | 功能     | 描述                                         | 优先级 |
| --------- | -------- | -------------------------------------------- | ------ |
| F-REQ-001 | 标准棋盘 | 15×15 格子，绘制格线，响应式尺寸             | P0     |
| F-REQ-002 | 落子     | 玩家轮流在交点落子，黑先白后                 | P0     |
| F-REQ-003 | 胜负判定 | 横、竖、两斜四向检测五子连珠，一方获胜即结束 | P0     |
| F-REQ-004 | 重新开始 | 对局结束后可清空棋盘重新开始                 | P0     |

### 2.2 多端与交互

| 编号      | 功能       | 描述                                                | 优先级 |
| --------- | ---------- | --------------------------------------------------- | ------ |
| F-REQ-005 | PC 落子    | 鼠标点击交点落子，坐标换算为 (row,col)              | P0     |
| F-REQ-006 | 触摸落子   | 移动端 touch 事件，与点击统一为「落点 → (row,col)」 | P0     |
| F-REQ-007 | 响应式棋盘 | 随视口/容器变化，格子大小按比例缩放                 | P0     |

### 2.3 AI（分阶段）

| 阶段    | 行为                      | 接口/说明                         |
| ------- | ------------------------- | --------------------------------- |
| Phase 1 | 随机或简单规则 AI         | 实现 `IAgent`，可替换为 Phase 2/3 |
| Phase 2 | Minimax + Alpha-Beta 剪枝 | 同接口，替换实现                  |
| Phase 3 | 预留 AlphaZero 风格接口   | 神经网络 + MCTS，接口预留         |

### 2.4 棋谱与复盘

| 编号      | 功能     | 描述                               | 优先级 |
| --------- | -------- | ---------------------------------- | ------ |
| F-REQ-008 | 保存棋谱 | 对局可导出为 SGF 或项目自定义 JSON | P1     |
| F-REQ-009 | 加载棋谱 | 从文件/存储加载并恢复棋盘状态      | P1     |
| F-REQ-010 | 复盘     | 按步数回放，支持前进/后退/暂停     | P1     |

### 2.5 小程序预留

| 编号      | 需求           | 描述                                                    |
| --------- | -------------- | ------------------------------------------------------- |
| F-REQ-011 | 逻辑与视图分离 | 核心逻辑不依赖 DOM，可在小程序逻辑层复用                |
| F-REQ-012 | 适配接口       | 预留微信小程序/小游戏的适配层接口（如渲染、存储、事件） |

---

## 3. 非功能需求

- **性能**：首屏与落子反馈 &lt; 100ms，AI 响应（Phase 2）&lt; 3s。
- **兼容**：Chrome / Safari / 微信内置浏览器近期两版。
- **无障碍**：支持键盘操作（可选）、焦点与语义（可选）。

---

## 4. 验收标准

- 满足上述 P0 功能且在 Chrome 桌面、Safari 移动端、微信内可完成一轮完整对局（含重新开始）。
- 坐标换算与胜负判定通过单元测试覆盖。
- 非法落子（已有子、非己方回合）时给出提示且状态不更新；悔棋后 `board` 与 `history` 一致。
